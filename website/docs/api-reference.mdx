# API Reference

The `pond-ts` docs are intentionally split into two layers:

- the hand-written guides in this site, which explain the model and show real workflows
- the generated API reference, which gives you exact signatures, options, and return types

Use the generated reference when you want to answer questions like:

- What does `TimeSeries.aggregate(...)` accept?
- Which smoothing modes are available?
- What shape does `TimeSeries.fromJSON(...)` expect?
- Which methods live on `Event`, `Sequence`, or `BoundedSequence`?

Start with these core types:

- `TimeSeries`
- `Event`
- `Time`, `TimeRange`, and `Interval`
- `Sequence` and `BoundedSequence`

[Open the generated API reference](/api)

The reference is generated from the current exported TypeScript surface during the docs build, so it stays aligned with released code.
