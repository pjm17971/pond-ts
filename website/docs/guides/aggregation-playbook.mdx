# Aggregation Playbook

This guide covers aggregation end-to-end: bucket semantics, built-in reducers, custom reducers,
empty bucket behavior, and how aggregation relates to rolling windows.

## Core shape

Use `aggregate(sequence, mapping, options?)` to produce an interval-keyed `TimeSeries`.

```ts
import { Sequence, TimeRange } from 'pond-ts';

const aggregated = series.aggregate(
  Sequence.every('5m'),
  {
    cpu: 'avg',
    requests: 'sum',
    host: 'last',
  },
  { range: new TimeRange({ start, end }) },
);
```

The result key is always `interval`, with one output event per bucket.

`mapping` supports two shapes:

- shorthand by source column name:
  - `{ latency: 'avg', host: 'last' }`
- explicit output specs:
  - `{ latency_avg: { from: 'latency', using: 'avg' } }`

## Bucket membership rules

Aggregation uses half-open intervals:

- `[begin, end)`

So an event exactly at `end` belongs to the next bucket.

For interval-like source keys (`Interval` / `TimeRange`), a source event contributes to every
bucket it overlaps under half-open overlap rules.

## Built-in reducers

Pond ships 12 named reducers plus percentile patterns. See the
[Reducer Reference](./reducer-reference.mdx) for detailed behavior, empty-bucket
semantics, and rolling window complexity for each one.

**Numeric:** `sum`, `avg`, `min`, `max`, `median`, `stdev`, `difference`, `count`

**Percentiles:** `p50`, `p95`, `p99`, `p99.9`, etc. (any `p` + number 0--100)

**Any type:** `first`, `last`, `keep`

Typical mapping:

```ts
const byBucket = series.aggregate(Sequence.every('1m'), {
  value: 'avg',
  status: 'last',
  samples: 'count',
});
```

## Custom reducers

You can supply a reducer function per mapped column.

Reducer contract:

- input: `ReadonlyArray<ScalarValue | undefined>`
- output: `ScalarValue | undefined`

```ts
const custom = series.aggregate(Sequence.every('10m'), {
  value: (values) =>
    values.reduce<number>(
      (sum, value) => sum + (typeof value === 'number' ? value : 0),
      0,
    ),
  status: (values) =>
    values
      .filter((value): value is string => typeof value === 'string')
      .join(','),
});
```

Custom reducers are also supported in `rolling(...)` mappings.

## Named output columns (`from` + `using`)

When you want multiple outputs from one source column, use the explicit spec form:

```ts
const quantiles = series.aggregate(Sequence.every('5m'), {
  p50: { from: 'latency', using: 'p50', kind: 'number' },
  p95: { from: 'latency', using: 'p95', kind: 'number' },
  host: { from: 'host', using: 'last' },
});
```

Use `kind` for custom reducers when the output type differs from the source column.

## Empty-bucket behavior

When a bucket has no contributing source values:

- `count` and `sum` return `0`
- all other built-ins return `undefined`
- custom reducer return depends on your implementation

This makes it easy to choose between sparse semantics (`undefined`) and explicit zero semantics.

## Whole-series reduction with `reduce`

`reduce(...)` collapses an entire series to a scalar or record instead of bucketing by time.
It uses the same reducer specs as `aggregate` — built-in names or custom functions.

Single-column form returns a scalar:

```ts
const avg = series.reduce('cpu', 'avg'); // number | undefined
const total = series.reduce('requests', 'sum'); // number | undefined
```

Multi-column form returns a record:

```ts
const summary = series.reduce({
  cpu: 'avg',
  requests: 'sum',
  host: 'last',
});
// { cpu: number | undefined, requests: number | undefined, host: string | undefined }
```

Named output specs work here too:

```ts
const quantiles = series.reduce({
  p50: { from: 'latency', using: myP50Reducer, kind: 'number' },
  p99: { from: 'latency', using: myP99Reducer, kind: 'number' },
});
```

Empty-bucket semantics apply: `avg`/`min`/`max`/`first`/`last` return `undefined` on an
empty series, while `count` returns `0` and `sum` returns `0`.

## Grouping with `groupBy`

`groupBy(column)` partitions a series by the distinct values of a payload column,
returning `Map<string, TimeSeries<S>>`. Each group preserves the full schema and event order.

```ts
const perHost = series.groupBy('host');
// Map { "api-1" => TimeSeries, "api-2" => TimeSeries, ... }
```

An optional transform callback is applied to each group, avoiding intermediate maps:

```ts
const perHostAvg = series.groupBy('host', (group) =>
  group.reduce({ cpu: 'avg', requests: 'sum' }),
);
// Map { "api-1" => { cpu: 25, requests: 1200 }, ... }
```

This composes naturally with any series method:

```ts
// per-host hourly aggregation
const hourly = series.groupBy('host', (group) =>
  group.aggregate(Sequence.every('1h'), { cpu: 'avg' }),
);

// per-host rolling averages
const rolling = series.groupBy('host', (group) =>
  group.rolling('5m', { cpu: 'avg' }),
);
```

The transform callback also receives the group key as its second argument.

## Differences and rates with `diff` and `rate`

`diff(columns)` computes the per-event difference for one or more numeric columns.
`rate(columns)` divides the difference by the time gap in seconds, giving a per-second rate of change.

```ts
const deltas = series.diff('requests');
const perSecond = series.rate('requests');
```

Multiple columns:

```ts
const deltas = series.diff(['cpu', 'mem']);
const perSecond = series.rate(['cpu', 'mem']);
```

The first event gets `undefined` in affected columns (no previous value to diff against).
Non-targeted columns pass through unchanged.

To drop the first event instead of keeping it with `undefined`:

```ts
const deltas = series.diff('requests', { drop: true });
```

`rate` handles zero time gaps by returning `undefined` (avoids division by zero).

These compose naturally with `groupBy`:

```ts
const perHostRate = series.groupBy('host', (group) => group.rate('requests'));
```

## Percentage change with `pctChange`

`pctChange(columns)` computes `(curr - prev) / prev` for named numeric columns. The first
event gets `undefined` (no previous value). Time gap does not matter — purely value-relative.

```ts
const pct = series.pctChange('price');
const pct = series.pctChange(['cpu', 'mem']);
const pct = series.pctChange('price', { drop: true });
```

Returns `undefined` when the previous value is zero (avoids division by zero).

## Running accumulation with `cumulative`

`cumulative(spec)` computes running values for named numeric columns. Non-accumulated
columns pass through unchanged.

Built-in accumulators: `"sum"`, `"max"`, `"min"`, `"count"`. Custom accumulators take
`(acc: number, value: number) => number`.

```ts
const running = series.cumulative({ requests: 'sum' });
const peaks = series.cumulative({ cpu: 'max' });
const mixed = series.cumulative({
  requests: 'sum',
  cpu: 'max',
  errors: 'count',
});

// custom accumulator
const product = series.cumulative({ value: (acc, v) => acc * v });
```

When a source value is `undefined`, the accumulator carries forward its previous value.

## Lag/lead with `shift`

`shift(columns, n)` moves column values forward (positive = lag) or backward (negative = lead)
by N events. Vacated positions get `undefined`. Event keys are unchanged.

```ts
const lagged = series.shift('value', 1); // lag by 1
const lead = series.shift('value', -1); // lead by 1
const lagged = series.shift(['cpu', 'mem'], 2);
```

For time-based shifting on irregular data, align to a regular grid first, then shift by the
corresponding number of events.

## Gap-filling with `fill`

`fill(strategy, options?)` replaces `undefined` values in payload columns. A single strategy
applies to all columns, or a per-column mapping targets specific columns.

Strategies: `"hold"` (forward fill), `"bfill"` (backward fill), `"linear"` (time-interpolated),
`"zero"`.

```ts
// forward fill all columns
const filled = series.fill('hold');

// backward fill
const filled = series.fill('bfill');

// per-column strategies
const filled = series.fill({ cpu: 'linear', host: 'hold' });

// literal fill values (non-string values in the mapping)
const filled = series.fill({ cpu: 0, host: 'unknown' });
```

`limit` caps consecutive fills per column:

```ts
const filled = series.fill('hold', { limit: 3 });
```

`"linear"` interpolates by time between the nearest known values on each side. Leading and
trailing `undefined` runs are left unfilled (no extrapolation). `"hold"` leaves leading
`undefined` unfilled (no previous value to carry). `"bfill"` leaves trailing `undefined`
unfilled (no future value to propagate).

Typical pipeline after aggregation with empty buckets:

```ts
const hourly = series
  .aggregate(Sequence.every('1h'), { cpu: 'avg', host: 'last' })
  .fill({ cpu: 'linear', host: 'hold' });
```

## Method comparison

| Method                         | Purpose                                               |
| ------------------------------ | ----------------------------------------------------- |
| `aggregate(sequence, mapping)` | Group by fixed time buckets, output interval keys     |
| `rolling(window, mapping)`     | Moving values anchored per source event               |
| `reduce(column, reducer)`      | Collapse entire series to a scalar or record          |
| `groupBy(column)`              | Partition by column value, apply any method per group |
| `diff(columns)`                | Per-event differences                                 |
| `rate(columns)`                | Per-second rates of change                            |
| `pctChange(columns)`           | Percentage change relative to previous value          |
| `cumulative(spec)`             | Running sum, max, min, count, or custom accumulation  |
| `shift(columns, n)`            | Lag or lead column values by N events                 |
| `fill(strategy)`               | Gap-fill with hold, bfill, linear, zero, or literal   |
