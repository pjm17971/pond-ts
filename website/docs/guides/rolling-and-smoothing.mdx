# Rolling Windows and Smoothing

## Rolling windows

Use `rolling(...)` for windowed calculations over ordered events.

```ts
const rolling = series.rolling('5m', {
  cpu: 'avg',
  requests: 'sum',
});
```

This keeps the original key type and computes derived values at each event anchor.

You can also compute rolling values on a sequence grid:

```ts
const rolledOnGrid = series.rolling(Sequence.every('1m'), '5m', {
  cpu: 'avg',
});
```

Supported alignments:

- `trailing`
- `leading`
- `centered`

### Custom reducers

Rolling windows accept the same custom reducer functions as `aggregate(...)`. The
reducer contract is `(values: ReadonlyArray<ScalarValue | undefined>) => ScalarValue | undefined`.

```ts
const rolled = series.rolling('5m', {
  cpu: (values) => {
    const nums = values.filter((v): v is number => typeof v === 'number');
    return nums.length === 0
      ? undefined
      : nums.reduce((a, b) => a + b, 0) / nums.length;
  },
});
```

You can mix built-in and custom reducers in the same mapping:

```ts
const mixed = series.rolling('5m', {
  cpu: 'avg',
  status: (values) =>
    values.filter((v): v is string => typeof v === 'string').join(','),
});
```

## Smoothing

Use `smooth(...)` for trend extraction or visualization-friendly derived values.

### Choosing a method

- Use `ema` when you want the lightest-weight smoother and a good default for
  streaming-style trend lines.
- Use `movingAverage` when you want explicit windowed smoothing with predictable
  trailing, leading, or centered behavior.
- Use `loess` when shape fidelity matters more than speed and you want a more
  flexible offline smoother.

### EMA

```ts
const smoothed = series.smooth('cpu', 'ema', {
  alpha: 0.2,
  output: 'cpuTrend',
});
```

### Moving average

```ts
const averaged = series.smooth('cpu', 'movingAverage', {
  window: '5m',
  alignment: 'centered',
  output: 'cpuAvg',
});
```

### LOESS

```ts
const loess = series.smooth('cpu', 'loess', {
  span: 0.75,
  output: 'cpuLoess',
});
```
