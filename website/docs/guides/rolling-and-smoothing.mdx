# Rolling Windows and Smoothing

## Rolling windows

Use `rolling(...)` for windowed calculations over ordered events.

```ts
const rolling = series.rolling('5m', {
  cpu: 'avg',
  requests: 'sum',
});
```

This keeps the original key type and computes derived values at each event anchor.

You can also compute rolling values on a sequence grid:

```ts
const rolledOnGrid = series.rolling(Sequence.every('1m'), '5m', {
  cpu: 'avg',
});
```

Supported alignments:

- `trailing`
- `leading`
- `centered`

### Custom reducers

Rolling windows accept the same custom reducer functions as `aggregate(...)`. The
reducer contract is `(values: ReadonlyArray<ScalarValue | undefined>) => ScalarValue | undefined`.

```ts
const rolled = series.rolling('5m', {
  cpu: (values) => {
    const nums = values.filter((v): v is number => typeof v === 'number');
    return nums.length === 0
      ? undefined
      : nums.reduce((a, b) => a + b, 0) / nums.length;
  },
});
```

You can mix built-in and custom reducers in the same mapping:

```ts
const mixed = series.rolling('5m', {
  cpu: 'avg',
  status: (values) =>
    values.filter((v): v is string => typeof v === 'string').join(','),
});
```

### Multiple reducers over the same source column

For cases like "rolling avg **and** stdev of `cpu` in one pass" —
handy for anomaly-detection bands on a chart — pass an
`AggregateOutputMap` where each entry is `{ from, using, kind? }`.
This is the same shape `aggregate(...)` accepts, and you can mix
plain `{ col: reducer }` entries with spec entries in the same call:

```ts
const bands = series.rolling('1m', {
  avg: { from: 'cpu', using: 'avg' },
  sd: { from: 'cpu', using: 'stdev' },
});

// Feed a ±1.7σ band chart:
bands.toArray().forEach((row) => {
  const avg = row.get('avg') as number | undefined;
  const sd = row.get('sd') as number | undefined;
  if (avg !== undefined && sd !== undefined) {
    drawBand({
      t: row.begin(),
      lower: avg - 1.7 * sd,
      upper: avg + 1.7 * sd,
    });
  }
});
```

Output column names come from the map's keys, not the source column —
you can rename freely (`cpuAvg: { from: 'cpu', using: 'avg' }`). The
`kind?` field is an override for cases where the output type can't be
inferred from the reducer (e.g. a custom reducer returning a string).

**Return types narrow per-spec** as of v0.5.5: numeric reducers give
`number | undefined`, source-preserving reducers (`first` / `last` /
`keep`) give the source column's type, and the source's first-column
kind is preserved — a `Time`-keyed input stays `Time`-keyed on
output. `unique` and `top${N}` narrow to `ArrayValue | undefined`
(kind-level); element-level narrowing to
`ReadonlyArray<string>` is only available on `reduce(...)` today —
use a cast (or reach for `reduce` on a `tail`-sliced snapshot) if you
need that extra step.

The same narrowing applies to `aggregate(...)` when called with the
`AggregateOutputMap` shape — both now thread the spec through.

## Smoothing

Use `smooth(...)` for trend extraction or visualization-friendly derived values.

### Choosing a method

- Use `ema` when you want the lightest-weight smoother and a good default for
  streaming-style trend lines.
- Use `movingAverage` when you want explicit windowed smoothing with predictable
  trailing, leading, or centered behavior.
- Use `loess` when shape fidelity matters more than speed and you want a more
  flexible offline smoother.

### EMA

```ts
const smoothed = series.smooth('cpu', 'ema', {
  alpha: 0.2,
  output: 'cpuTrend',
});
```

The first few events of an EMA output are noisy — the smoother starts
with the first raw value as its seed and needs ~`1/alpha` samples to
converge. Pass `warmup: N` to drop the first `N` rows from the output
once the smoother has warmed up. The dropped rows still feed the
smoother state, so the kept rows are computed against a warm EMA — it
only trims the output, not the computation:

```ts
// alpha=0.3 converges in ~3-4 samples; ask for 4 warmup rows to trim.
const smoothed = series.smooth('cpu', 'ema', {
  alpha: 0.3,
  warmup: 4,
});
// smoothed.length === series.length - 4 (or 0 if series is smaller)
```

`warmup: 0` is a no-op, and warmup values greater than the series
length produce an empty series (same schema, no rows).

### Moving average

```ts
const averaged = series.smooth('cpu', 'movingAverage', {
  window: '5m',
  alignment: 'centered',
  output: 'cpuAvg',
});
```

### LOESS

```ts
const loess = series.smooth('cpu', 'loess', {
  span: 0.75,
  output: 'cpuLoess',
});
```
