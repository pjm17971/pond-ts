# Sampling Overview

Two operations resample a `TimeSeries` onto a [`Sequence`](../concepts/overview#sequence)
grid. They look similar on the surface — both take a sequence and
return interval-keyed events — but they answer different questions.

- **`align(seq, opts)`** asks "**what was the value at this grid point?**"
  and returns one event per grid point.
- **`aggregate(seq, mapping)`** asks "**what's the rollup of every event
  inside this bucket?**" and returns one event per bucket, computed
  from a reducer mapping.

Pandas users: pond-ts splits what `.resample()` does into two
operators. `align` ≈ `.asfreq()` (plus `.ffill()` or `.interpolate()`);
`aggregate` ≈ `.resample().agg()`.

## Mental model

Source events sit on the timeline at their own (sometimes irregular)
times. Both `align` and `aggregate` emit one **interval-keyed** event
per grid step — same key shape, different value rule:

- `align` samples the source at a chosen point inside each grid step
  (`'begin'` / `'center'` / `'end'`), with the value either **held**
  from the most recent source ≤ that point or **linearly interpolated**
  between the two surrounding sources.
- `aggregate` collapses every source event whose key falls in
  `[step.begin, step.end)` through a reducer.

## Comparison

The same source, the same grid, two outputs:

```ts
import { Sequence, TimeSeries } from 'pond-ts';

const schema = [
  { name: 'time', kind: 'time' },
  { name: 'cpu', kind: 'number' },
] as const;

const cpu = TimeSeries.fromJSON({
  name: 'cpu',
  schema,
  rows: [
    ['2025-01-01T00:00:00Z', 0.31],
    ['2025-01-01T00:00:30Z', 0.44],
    ['2025-01-01T00:00:45Z', 0.52],
    ['2025-01-01T00:01:30Z', 0.48],
    ['2025-01-01T00:02:10Z', 0.63],
  ],
});

// align — one row per minute, value carried from the most recent source.
cpu.align(Sequence.every('1m'), { method: 'hold' });

// aggregate — one row per minute, value reduced from sources inside.
cpu.aggregate(Sequence.every('1m'), { cpu: 'avg' });
```

For a chart at one-minute resolution where every grid point should
have a value, `align` is right. For "average CPU per minute," `aggregate`
is right. The two are complementary, not interchangeable.

## `aggregate` quick reference

```ts
cpu.aggregate(Sequence.every('5s'), {
  cpu: 'avg',
  requests: 'sum',
  host: 'last',
});
```

- **Mapping shapes** — shorthand (`col: 'reducer'`) or explicit
  (`{ from, using, kind? }`). Mix freely; see
  [Aggregation → Mapping shapes](./aggregation#mapping-shapes).
- **Custom reducers** — pass a function instead of a name.
- **Buckets** are half-open `[begin, end)` — boundary events land
  in the **later** bucket.
- **`range`** — pin the output range explicitly (default: source
  extent).
- **Empty buckets** — `count` and `sum` emit `0`; most others emit
  `undefined`. See [Reducer reference →
  Empty-bucket behavior](./reducer-reference#empty-bucket-behavior).

For the deep dive — `LiveAggregation`, `reduce`,
`arrayAggregate` — see [Aggregation](./aggregation). For partition
ops (`groupBy`) and reshape (`pivotByGroup`, `join`), see
[Reshaping](./reshape).

## `align` quick reference

```ts
cpu.align(Sequence.every('5s'), {
  method: 'hold', // 'hold' (carry previous) | 'linear' (interpolate)
  sample: 'begin', // 'begin' | 'center' | 'end'
});
```

- **`method: 'hold'`** — carry the most recent source value at or
  before the sample point; output is `undefined` before any source
  is available.
- **`method: 'linear'`** — interpolate between the two neighboring
  sources. Throws on non-`Time`-keyed sources (`TimeRange` /
  `Interval`); falls back to `'hold'` for non-numeric columns.
- **`sample`** — where inside each grid step to take the snapshot:
  `'begin'` (interval start), `'center'` (midpoint), `'end'`
  (interval's exclusive boundary, i.e. the start of the next step).
- **`range`** — pin the output range explicitly; default is the
  source extent.

:::tip[Want time-keyed output for charting?]
`align` returns interval-keyed events (consistent with `aggregate`).
For point-keyed output — what most chart libraries expect — chain
`.asTime({ at })`:

```ts
const points = cpu
  .align(Sequence.every('5m'), { method: 'hold', sample: 'end' })
  .asTime({ at: 'end' });
// points.at(0)?.key() is now a Time at the interval's end.
```

`asTime`'s `at` option (`'begin' | 'center' | 'end'`) picks where on
each interval the resulting `Time` lands.
:::
