# Aggregation Playbook

How to compose `aggregate` with the rest of the library. For the
aggregate signature itself — mapping shapes, bucket membership, custom
reducers, empty buckets — see
[Align and Aggregate](./alignment-and-aggregation).

This page covers:

- [**`reduce()`**](#reduce) — collapse a whole series to a scalar or
  record. Bucket size: all of it.
- [**`groupBy()`**](#groupby) — partition by a column value, apply any
  method per group. The "second dimension" of aggregation.
- [**Per-event transforms**](#per-event-transforms) — `diff`, `rate`,
  `pctChange`, `cumulative`, `shift`. Stateless or tiny-state
  derivations over consecutive events.
- [**`fill()`**](#fill) — gap-fill the `undefined`s that aggregation
  and alignment produce.

Coming from pandas:

- `reduce` ≈ `.agg()` / a scalar on a `Series`
- `groupBy(col, fn)` ≈ `.groupby(col).apply(fn)`
- `diff` / `rate` / `pctChange` ≈ `.diff()` / rate-of-change / `.pct_change()`
- `cumulative` ≈ `.cumsum()` / `.cummax()` / etc.
- `shift` ≈ `.shift(n)`
- `fill('hold')` ≈ `.ffill()`; `fill('bfill')` ≈ `.bfill()`; `fill('linear')` ≈ `.interpolate('time')`

## `reduce`

Whole-series → scalar or record. Same reducer vocabulary as
`aggregate`, same empty-bucket behavior.

```ts
// Single-column form returns a scalar (wide ColumnValue type).
const avg = series.reduce('cpu', 'avg'); // ColumnValue | undefined
// Narrow at the call site when you know the reducer's output:
const avgN = avg as number | undefined;
```

:::tip[Type narrowing on `reduce`]
The single-column form returns the wide `ColumnValue | undefined` — the
type system can't see through a string reducer name. Use the
**mapping form** whenever you want narrowed output types at the call
site (see below). The mapping form reads slightly heavier but removes
the casts.
:::

Multi-column form returns a record. Types narrow per reducer (`avg` /
`sum` / `count` → `number | undefined`; `unique` → `ReadonlyArray<T> |
undefined`; `first` / `last` / `keep` preserve source kind):

```ts
const summary = series.reduce({
  cpu: 'avg',
  requests: 'sum',
  host: 'unique',
});
// summary.cpu      : number | undefined
// summary.requests : number | undefined
// summary.host     : ReadonlyArray<string> | undefined
```

Named outputs via the explicit spec form:

```ts
const quantiles = series.reduce({
  p50: { from: 'latency', using: 'p50' },
  p95: { from: 'latency', using: 'p95' },
  p99: { from: 'latency', using: 'p99' },
});
```

See [Reducer Reference](./reducer-reference#type-narrowing-on-reducemapping)
for the full type-narrowing story.

## `groupBy`

Partition by a column value. Returns `Map<string, TimeSeries<S>>` or
(with a transform callback) `Map<string, T>`.

```ts
const perHost = series.groupBy('host');
// Map<'api-1' | 'api-2' | ..., TimeSeries<S>>
```

The transform callback is the common form — it avoids building
intermediate maps of `TimeSeries`:

```ts
const perHostAvg = series.groupBy('host', (group) =>
  group.reduce({ cpu: 'avg', requests: 'sum' }),
);
// Map { 'api-1' => { cpu: 0.42, requests: 12543 }, ... }
```

The callback receives the group key as a second argument:

```ts
series.groupBy('host', (group, host) => ({
  host,
  avg_cpu: group.reduce('cpu', 'avg'),
}));
```

Composes with every other method:

```ts
// Per-host hourly aggregation.
const hourly = series.groupBy('host', (group) =>
  group.aggregate(Sequence.every('1h'), { cpu: 'avg' }),
);

// Per-host rolling averages.
const rolling = series.groupBy('host', (group) =>
  group.rolling('5m', { cpu: 'avg' }),
);

// Per-host outliers — one Map<host, anomaly series>.
const anomalies = series.groupBy('host', (group) =>
  group.outliers('cpu', { window: '1m', sigma: 2 }),
);
```

## Per-event transforms

Stateless or tiny-state derivations over consecutive events. All of
these preserve the event count and key type; they just append or
replace value columns.

### `diff(cols)` — per-event difference

```ts
const deltas = series.diff('requests');
// First event: requests = undefined (no previous).
// Subsequent: requests_i − requests_{i-1}.

const deltas = series.diff(['cpu', 'mem']);

// Drop the first-event undefined instead of keeping it:
const deltas = series.diff('requests', { drop: true });
```

### `rate(cols)` — delta per second

Divides `diff` by the time gap (in seconds). Per-second rate of change.

```ts
const rps = series.rate('requests');
// rps.at(i).get('requests') = (requests_i − requests_{i-1}) / gap_seconds
```

When two consecutive events share a timestamp (`dt = 0`), `rate`
returns `undefined` for that event rather than dividing by zero. If
your data has concurrent events and the numerator is meaningful, filter
per-producer first.

### `pctChange(cols)` — percentage change

`(curr − prev) / prev`. Purely value-relative — time gap doesn't
matter. Returns `undefined` when the previous value is 0.

```ts
const returns = prices.pctChange('price');
```

### `cumulative(spec)` — running values

Running sum / max / min / count, or a custom accumulator. Non-
accumulated columns pass through.

```ts
const running = series.cumulative({ requests: 'sum', cpu: 'max' });
const errors = series.cumulative({ errors: 'count' });

// Custom accumulator: (acc, value) => acc
const product = series.cumulative({ factor: (acc, v) => acc * v });
```

When a source value is `undefined`, the accumulator carries forward
its previous value — no NaN propagation.

### `shift(cols, n)` — lag / lead

Move column values forward (positive `n` = lag) or backward (negative
= lead) by `n` events. Vacated slots get `undefined`. Keys unchanged.

```ts
const lagged = series.shift('value', 1); // lag by 1
const lead = series.shift('value', -1); // lead by 1
const multi = series.shift(['cpu', 'mem'], 2);
```

For time-based shifts on irregular data, align to a regular grid
first, then shift by the corresponding number of events.

## `fill`

Replaces `undefined`s in value columns. The typical post-aggregation
cleanup.

### Strategies

- **`'hold'`** — forward-fill: carry the last known value. Leading
  `undefined` runs stay unfilled (nothing to carry).
- **`'bfill'`** — backward-fill: pull in the next known value.
  Trailing `undefined` runs stay unfilled.
- **`'linear'`** — time-interpolate between the two nearest known
  values on either side. Leading and trailing `undefined` runs stay
  unfilled (no extrapolation).
- **`'zero'`** — fill with `0`. Numeric columns only.

A single strategy applies to every value column:

```ts
series.fill('hold');
series.fill('linear');
series.fill('zero');
```

A per-column mapping targets specific columns; other columns pass
through with their `undefined`s intact:

```ts
series.fill({ cpu: 'linear', host: 'hold' });
```

Literal fill values (anything that's not a strategy string) set the
column to that value:

```ts
series.fill({ cpu: 0, host: 'unknown' });
```

### Caps

`limit` caps consecutive fills per column:

```ts
series.fill('hold', { limit: 3 });
// No more than 3 consecutive undefined values are filled from any
// one source event. Longer gaps retain undefined in the tail.
```

### Typical post-aggregate pipeline

```ts
const hourly = series
  .aggregate(Sequence.every('1h'), { cpu: 'avg', host: 'last' })
  .fill({ cpu: 'linear', host: 'hold' });
```

## Method comparison

| Method                         | Purpose                                               |
| ------------------------------ | ----------------------------------------------------- |
| `aggregate(sequence, mapping)` | Group by fixed time buckets → one output per bucket   |
| `align(sequence, options)`     | Resample onto a grid → one output per grid point      |
| `rolling(window, mapping)`     | Windowed values anchored per source event             |
| `reduce(mapping)`              | Collapse entire series to a scalar or record          |
| `groupBy(col, fn)`             | Partition by column value, apply any method per group |
| `diff(cols)`                   | Per-event differences                                 |
| `rate(cols)`                   | Per-second rates of change                            |
| `pctChange(cols)`              | Percentage change relative to previous value          |
| `cumulative(spec)`             | Running sum / max / min / count / custom              |
| `shift(cols, n)`               | Lag or lead column values by N events                 |
| `fill(strategy)`               | Gap-fill: hold / bfill / linear / zero / literal      |
